# Terraform VPC・ネットワークモジュール構築

## 背景

ECS (Fargate) + RDS (PostgreSQL) + ALB を配置するためのネットワーク基盤が必要だった。セキュリティとマルチAZ可用性を考慮した設計が求められた。

## 決定内容

- **VPC**: 環境ごとにCIDRを分離（dev: 10.0.0.0/16, prod: 10.1.0.0/16）
- **サブネット**: 2AZにパブリック・プライベートサブネットを各2つ配置
- **ルーティング**: パブリック→IGW、プライベート→NAT Gateway（単一AZ）
- **セキュリティグループ**: ALB（80/443受信）→ ECS（8000、ALBからのみ）→ RDS（5432、ECSからのみ）のチェーン構成
- **AZ取得**: `data.aws_availability_zones` で動的に取得し、ハードコードを回避

## 代替案

| 案 | 却下理由 |
|---|---------|
| NATゲートウェイを2AZに配置 | コスト倍増。dev環境では単一AZで十分 |
| AZをハードコード | リージョン変更時に修正が必要。動的取得の方が柔軟 |
| セキュリティグループをサービスごとに別モジュール | ネットワーク層で一元管理した方が依存関係が明確 |

## 結果

- マルチAZ構成で可用性を確保
- プライベートサブネットにECS/RDSを配置するセキュアな構成
- セキュリティグループのチェーンで最小権限のアクセス制御を実現
